Interface

ComputableDAGs.AbstractDevice — Type
AbstractDevice

Abstract base type for every device, like GPUs, CPUs or any other compute devices. Every implementation needs to implement various functions and needs a member cacheStrategy.

source
ComputableDAGs.CacheStrategy — Type
CacheStrategy

Abstract base type for caching strategies.

See also: strategies

source
ComputableDAGs.Dictionary — Type
Dictionary <: CacheStrategy

A caching strategy relying on a dictionary of Symbols to store every input and output.

Implements the CacheStrategy interface.

source
ComputableDAGs.LocalVariables — Type
LocalVariables <: CacheStrategy

A caching strategy relying solely on local variables for every input and output.

Implements the CacheStrategy interface.

source
ComputableDAGs.Machine — Type
Machine

A representation of a machine to execute on. Contains information about its architecture (CPUs, GPUs, maybe more). This representation can be used to make a more accurate cost prediction of a DAG state.

See also: Scheduler

source
ComputableDAGs.CACHE_STRATEGIES — Constant
CACHE_STRATEGIES::Dict{Type{AbstractDevice}, Symbol}

Global dictionary of available caching strategies per device. Each implementation of AbstractDevice should add its available strategies to the dictionary.

See also: strategies

source
ComputableDAGs.DEVICE_TYPES — Constant
DEVICE_TYPES::Vector{Type}

Global vector of available and implemented device types. Each implementation of a AbstractDevice should add its concrete type to this vector.

See also: device_types, get_devices

source
ComputableDAGs.cuda_kernel — Function
cuda_kernel(
    graph::DAG,
    instance,
    machine::Machine,
    context_module::Module
)

Return a function of signature compute_<id>(input::CuVector, output::CuVector, n::Int64), which will return the result of the DAG computation of the input on the given output variable.

Note

This function is only available when the CUDA Extension is loaded by using CUDA before using ComputableDAGs

source
ComputableDAGs.default_strategy — Function
default_strategy(deviceType::Type{T}) where {T <: AbstractDevice}

Interface function that must be implemented for every subtype of AbstractDevice. Returns the default CacheStrategy to use on the given device type. See also: cache_strategy, set_cache_strategy

source
ComputableDAGs.gen_access_expr — Function
gen_access_expr(device::AbstractDevice, symbol::Symbol)

Interface function that must be implemented for every subtype of AbstractDevice and at least one CacheStrategy. Return an Expr or QuoteNode accessing the variable identified by [symbol].

source
ComputableDAGs.gen_cache_init_code — Function
gen_cache_init_code(device::AbstractDevice)

Interface function that must be implemented for every subtype of AbstractDevice and at least one CacheStrategy. Returns an Expr initializing this device's variable cache.

The strategy is a symbol

source
ComputableDAGs.get_devices — Function
get_devices(t::Type{T}; verbose::Bool) where {T <: AbstractDevice}

Interface function that must be implemented for every subtype of AbstractDevice. Returns a Vector{Type} of the devices for the given AbstractDevice Type available on the current machine.

source
ComputableDAGs.measure_device! — Function
measure_device!(device::AbstractDevice; verbose::Bool)

Interface function that must be implemented for every subtype of AbstractDevice. Measures the compute speed of the given device and writes into it.
